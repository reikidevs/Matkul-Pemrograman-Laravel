### 2.2 Activity Diagram

Activity Diagram menggambarkan alur proses bisnis dalam sistem. Diagram ini menunjukkan langkah-langkah yang dilakukan dari awal hingga akhir proses.

#### 2.2.1 Activity Diagram - [Nama Proses 1]

[Sisipkan Gambar Activity Diagram di sini]

**Gambar 2.2 Activity Diagram - [Nama Proses 1]**

**Penjelasan Alur:**

1. [Langkah 1]: [Penjelasan]
2. [Langkah 2]: [Penjelasan]
3. [Decision Point]: [Penjelasan kondisi]
   - Jika [kondisi A]: [Langkah selanjutnya]
   - Jika [kondisi B]: [Langkah alternatif]

**Contoh:**

**Penjelasan Alur Proses Peminjaman Buku:**

1. **Mulai**: Anggota membuka aplikasi
2. **Login**: Anggota memasukkan NIM dan password

   - Jika valid: Lanjut ke dashboard
   - Jika tidak valid: Tampil pesan error, kembali ke login
4. **Cari Buku**: Anggota mencari buku yang ingin dipinjam
5. **Pilih Buku**: Anggota memilih buku dari hasil pencarian
6. **Cek Ketersediaan**: Sistem mengecek stok buku
   - Jika tersedia: Lanjut ke proses peminjaman
   - Jika tidak tersedia: Tampil pesan "Buku tidak tersedia"
7. **Cek Kuota**: Sistem mengecek jumlah buku yang sedang dipinjam
   - Jika < 3 buku: Lanjut
   - Jika = 3 buku: Tampil pesan "Kuota penuh"
8. **Konfirmasi Peminjaman**: Anggota mengkonfirmasi peminjaman
9. **Buat Transaksi**: Sistem membuat transaksi peminjaman (status: pending)
10. **Notifikasi**: Sistem mengirim notifikasi ke pustakawan
11. **Selesai**

### 2.3 Sequence Diagram

Sequence Diagram menggambarkan interaksi antar objek dalam sistem berdasarkan urutan waktu. Diagram ini menunjukkan bagaimana objek-objek berkomunikasi satu sama lain.

#### 2.3.1 Sequence Diagram - [Nama Skenario 1]

[Sisipkan Gambar Sequence Diagram di sini]

**Gambar 2.4 Sequence Diagram - [Nama Skenario 1]**

**Objek yang Terlibat:**

1. **[Nama Objek 1]**: [Deskripsi peran]
2. **[Nama Objek 2]**: [Deskripsi peran]
3. **[Nama Objek 3]**: [Deskripsi peran]
4. **[Nama Objek 4]**: [Deskripsi peran]

**Alur Interaksi:**

1. [Objek A] mengirim pesan [nama pesan] ke [Objek B]
2. [Objek B] memproses dan mengirim [nama pesan] ke [Objek C]
3. [Objek C] melakukan [aksi] dan mengembalikan [hasil] ke [Objek B]
4. [Objek B] mengirim [response] ke [Objek A]

**Contoh:**

**Objek yang Terlibat:**

1. **Anggota**: Pengguna yang melakukan peminjaman
2. **BukuPage**: Halaman interface untuk memilih buku
3. **PeminjamanController**: Controller yang menangani logika peminjaman
4. **Buku**: Model yang merepresentasikan data buku
5. **Peminjaman**: Model yang merepresentasikan transaksi peminjaman
6. **Database**: Penyimpanan data

**Alur Interaksi:**

1. Anggota memilih buku dan klik tombol "Pinjam"
2. BukuPage mengirim request peminjaman ke PeminjamanController
3. PeminjamanController meminta Buku untuk mengecek ketersediaan
4. Buku melakukan query ke Database untuk mengecek stok
5. Database mengembalikan data stok ke Buku
6. Buku mengembalikan status ketersediaan ke PeminjamanController
7. PeminjamanController membuat transaksi baru melalui Peminjaman
8. Peminjaman menyimpan data ke Database
9. Database mengembalikan konfirmasi sukses
10. PeminjamanController mengirim response sukses ke BukuPage
11. BukuPage menampilkan pesan konfirmasi ke Anggota

#### 2. Class [Nama Class 2]

[Penjelasan seperti di atas]

**Relasi Antar Class:**

1. **[Class A] → [Class B]**: [Jenis relasi] - [Penjelasan]
2. **[Class C] → [Class D]**: [Jenis relasi] - [Penjelasan]

**Contoh:**

**Penjelasan Class:**

#### 1. Class User

**Atribut:**
- id: int - Primary key, auto increment
- email: string - Email pengguna (unique)
- password: string - Password terenkripsi
- role: enum - Peran pengguna (anggota, pustakawan, admin)
- created_at: datetime - Waktu pembuatan akun
- updated_at: datetime - Waktu update terakhir

**Method:**
- login(): boolean - Melakukan proses login
- logout(): void - Melakukan proses logout
- changePassword(): boolean - Mengubah password

#### 2. Class Anggota (extends User)

**Atribut:**
- nim: string - Nomor Induk Mahasiswa (unique)
- nama: string - Nama lengkap mahasiswa
- jurusan: string - Jurusan mahasiswa
- no_hp: string - Nomor handphone
- alamat: text - Alamat lengkap

**Method:**
- pinjamBuku(): boolean - Melakukan peminjaman buku
- perpanjangPinjaman(): boolean - Memperpanjang masa pinjam
- lihatRiwayat(): array - Melihat riwayat peminjaman
- lihatDenda(): decimal - Melihat total denda

#### 3. Class Buku

**Atribut:**
- id: int - Primary key
- isbn: string - ISBN buku (unique)
- judul: string - Judul buku
- penulis: string - Nama penulis
- penerbit: string - Nama penerbit
- tahun: int - Tahun terbit
- kategori: string - Kategori buku
- stok: int - Jumlah stok tersedia
- lokasi: string - Lokasi rak buku

**Method:**
- cekKetersediaan(): boolean - Mengecek apakah buku tersedia
- kurangiStok(): void - Mengurangi stok saat dipinjam
- tambahStok(): void - Menambah stok saat dikembalikan

#### 4. Class Peminjaman

**Atribut:**
- id: int - Primary key
- anggota_id: int - Foreign key ke tabel anggotas
- buku_id: int - Foreign key ke tabel bukus
- tanggal_pinjam: date - Tanggal peminjaman
- tanggal_kembali: date - Tanggal jatuh tempo
- tanggal_dikembalikan: date - Tanggal pengembalian aktual (nullable)
- status: enum - Status peminjaman (pending, approved, dipinjam, dikembalikan)
- denda: decimal - Jumlah denda (default 0)

**Method:**
- approve(): void - Menyetujui peminjaman
- perpanjang(): boolean - Memperpanjang masa pinjam
- kembalikan(): void - Proses pengembalian buku
- hitungDenda(): decimal - Menghitung denda keterlambatan

**Relasi Antar Class:**

1. **User → Anggota**: Inheritance (Generalization) - Anggota mewarisi atribut dan method dari User
2. **Anggota → Peminjaman**: One to Many - Satu anggota dapat memiliki banyak peminjaman
3. **Buku → Peminjaman**: One to Many - Satu buku dapat dipinjam berkali-kali (dalam waktu berbeda)
4. **Peminjaman → Anggota**: Many to One - Banyak peminjaman dimiliki oleh satu anggota
5. **Peminjaman → Buku**: Many to One - Banyak peminjaman merujuk ke satu buku
